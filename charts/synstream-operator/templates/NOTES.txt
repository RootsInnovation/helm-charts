1. SynStream Operator has been deployed!

2. Check the operator status:
   kubectl get deployment {{ include "synstream-operator.fullname" . }} -n {{ .Release.Namespace }}

3. Watch for the operator to be ready:
   kubectl wait --for=condition=available deployment/{{ include "synstream-operator.fullname" . }} -n {{ .Release.Namespace }} --timeout=300s

4. Verify CRDs are installed:
   kubectl get crd flows.synstream.rtsinv.com

{{- if .Values.metrics.enabled }}
5. Metrics are available at:
   kubectl port-forward service/{{ include "synstream-operator.fullname" . }}-metrics-service {{ .Values.metrics.bindAddress }} -n {{ .Release.Namespace }}
{{- end }}

6. Create your first SynStream Flow:
   kubectl apply -f - <<EOF
   apiVersion: synstream.rtsinv.com/v1
   kind: Flow
   metadata:
     name: my-first-flow
     namespace: {{ .Release.Namespace }}
   spec:
     image:
       url: {{ include "synstream-operator.defaultImageURL" . }}
       tag: {{ include "synstream-operator.defaultImageTag" . }}
     replicas: 1
     service:
       type: ClusterIP
       port: 1880
       targetPort: 1880
   EOF

7. Check your Flow status:
   kubectl get flows -n {{ .Release.Namespace }}
   kubectl describe flow my-first-flow -n {{ .Release.Namespace }}

For more information, visit: https://github.com/RootsInnovation/synstream-operator
